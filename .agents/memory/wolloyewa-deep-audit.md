## Dispatcher double-registration bug
`/search` and `/checkout` were registered both as standalone CommandHandlers AND as ConversationHandler entry points. The standalone handler always fires first (registered earlier in group 0), so the ConversationHandler was never entered via commands. **Fix:** Remove the standalone registrations; only the ConversationHandler entry_points remain.

**How to apply:** Never register the same command as both a standalone CommandHandler and a ConversationHandler entry_point in the same group.

## Missing callback registrations in dispatcher
Many callback patterns generated by handler code were never registered:
- `add_to_cart_*` → `cart.cart_callback`
- `add_to_wishlist_*` → `wishlist.wishlist_callback`
- `cat_page_*` → new `catalog.category_page_callback`
- `wishlist_*` → `wishlist.wishlist_callback`
- `search_*`, `sort_*`, `filter_*` → `search.filter_callback`

**Why:** Handlers generate buttons with these data strings but they were never wired in dispatcher. Buttons would silently do nothing.

## `cat_page_` parsing crash in catalog.py
`category_callback` registered for `^cat_` pattern also matched `cat_page_<cat_id>_<page>`. Then `int(query.data.split("_")[1])` → `int("page")` → ValueError crash.

**Fix:** Register a dedicated `category_page_callback` for `^cat_page_` BEFORE the generic `^cat_` handler, using `parts = query.data.split("_"); category_id=int(parts[2]); page=int(parts[3])`.

## ConversationHandler entry-point conflict with ^menu_ handler
`^menu_` handler is registered globally before ConversationHandlers. So `^menu_search$` and `^menu_feedback$` in ConversationHandler entry_points were shadowed. The menu_callback in start.py calls search_command/feedback_command (show the prompt) but doesn't set conv state. Accepted limitation — /search and /feedback commands still enter conversations correctly.

## web_app templates directory missing
`bot/web_app/router.py` uses `Jinja2Templates(directory=...)` pointing to `bot/web_app/templates/` which didn't exist. Created `bot/web_app/templates/index.html` with a basic product listing Mini App page.
